Skills

JavaScript & Node.js

Twitterライクな575を投稿するSNSアプリケーションをフロントエンドはJavaScirpt、バックエンドは
Node.jsで開発。フロントはReactなどのフレームワークを使用せずに全てValilaJSで記述しました。
サーバー側はexpressフレームワークを使用してCRUD操作からmongo DB、外部クラウドストレージとの接続、API作成など基本的なことはできるようにしています。

TypeScript

このポートフォリオサイトの作成にTypeScriptを使用していますがそれ以外はチュートリアルをいくつかこなした程度です。 しかし、今後の個人開発では全てフロントもバックエンドもJavaScriptからTypeScriptに移行する予定です。

MongoDB

575俳句投稿アプリではデータベースにmongoDBを使用。使用したきっかけはチュートリアルや教材等ではNode.jsとMongoDBの組み合わせが多かった為。Node.jsではmongooseのライブラリを使用してwebホスティングサービスのAtlasと連携。バックエンドでNode.jsと組み合わせるならMySQLよりnode.jsを使用したい。

HTML/CSS

基本的なことであればフレームワークなしでHTMLとCSSを使ってwebサイトをデザインできます。

Python & Flask

Pythonを使用してデータベースから取得した文章を自動で取得しツイートするアプリケーションを開発したことがあります。使用したDBはsqlite3。
その他、複数の連番画像からスプライトシートを作成するプログラム / CSVから文章を取得後自動ツイートするbot / passwordを生成するプログラム/webスクレイピング 等

MySQL

基本的なSQL文は記述可能。アプリケーション内に組み込んでNode.jsと接続したことはチュートリアル以外ではまだありません。

Git

個人開発で使用する範囲のみ使用経験あり。基本的に全てのプロジェクトはGitで管理。branchを作ってmergeしたり現場を想定しながら作業しています。

Webpack

このポートフォリオサイトはwebpackを使用しています。動的なwebアプリケーションでは現状難しい為使用していません。

Docker

575アプリケーションをGCPのCloud Runにデプロイする際にコンテナが必要なためDockerを導入。docker-composeを使って開発環境を複数作ることができる。しかし、M1 MacではDockerでbuildしたimageがアップロードできなかった為gcloud buildsを使用したクラウドビルドを使用。

GCP/Cloud Run

WebアプリケーションのデプロイにGoogle cloud platformのCloud Runを使用。Cliからコンテナイメージをビルドし自動的にCloud Runにアップロードできる。

GCP/Cloud Storage

575アプリケーションで投稿する画像は全てCloud Storageで保管、管理している。Cloud StorageのAPIを使用いてアプリケーションとmongo DBと連携している。
